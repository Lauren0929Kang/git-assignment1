## top — 실시간 프로세스 모니터링


실행 중인 프로세스를 실시간으로 보여주는 명령어로 윈도우의 작업 관리자와 비슷하다.

- 화면 상단: 시스템 요약(가동 시간, 부하 평균, CPU/메모리 사용량)을 보여줌
- 화면 하단: 프로세스 목록 (CPU 사용률 순 정렬, 1초 간격 자동 갱신)을 보여줌

### 실행 중 단축키

| 키 | 동작 |
|----|------|
| `q` | 종료 |
| `P` | CPU 사용률 순 정렬 (대문자) |
| `M` | 메모리 사용률 순 정렬 |
| `k` | 특정 PID에 시그널 보내기 |
| `1` | CPU 코어별 사용량 개별 표시 |
| `u` | 특정 사용자의 프로세스만 보기 |

## ps — 프로세스 스냅샷

`top`이 실시간이라면, `ps`는 특정 순간의 프로세스 상태를 한 번 찍어 보여준다.



옵션 의미:

- `a` — 모든 사용자의 프로세스
- `u` — 사용자 친화적 형식 (소유자·CPU·메모리 사용률 표시)
- `x` — 터미널에 연결되지 않은 프로세스도 포함


---

## jobs — 현재 셸의 백그라운드 작업

`top`/`ps`가 시스템 전체를 본다면, `jobs`는 **현재 터미널 세션의** 백그라운드/일시정지 작업만 보여준다.



### 작업 제어(job control) 관련 명령

| 명령 | 동작 |
|------|------|
| `Ctrl + Z` | 실행 중인 작업을 일시정지(suspend) |
| `bg %1` | 1번 작업을 백그라운드에서 재개 |
| `fg %1` | 1번 작업을 포그라운드로 가져오기 |
| `jobs -l` | PID까지 함께 표시 |

## kill — 프로세스에 시그널 보내기

프로세스에 시그널 을 보내는 명령으로 기본 동작이 종료라서 이러한 이름이 붙었다.



### 자주 쓰는 시그널

| 시그널 | 번호 | 의미 |
|--------|------|------|
| `SIGTERM` | 15 | 정상 종료 요청 (**기본값**). 정리 작업 후 종료 |
| `SIGKILL` | 9 | 강제 종료. 무시 불가, 정리 작업 없이 즉시 종료 |
| `SIGHUP` | 1 | 재시작 / 설정 재로드에 자주 사용 |
| `SIGINT` | 2 | 인터럽트 (`Ctrl + C`와 동일) |

### 사용 예시

```bash
kill -15 1234        # 정상 종료 (기본값과 동일)
kill -9 1234         # 강제 종료
kill -SIGKILL 1234   # 이름으로 지정해도 됨
```


### 이름으로 종료하기

```bash
pkill firefox        # 이름으로 프로세스 종료
killall firefox      # 같은 이름의 프로세스 전부 종료
kill %1              # jobs의 작업 번호로도 가능
```

##  4가지 명령어의 연결

```
top / ps  →  문제 프로세스를 찾아 PID 확인
                      ↓
                   kill  →  해당 PID에 종료 시그널 전송

jobs  →  내가 직접 띄운 백그라운드 작업을 관리 (좁은 범위)
```

1. `top` / `ps` 로 문제가 되는 프로세스를 찾아 **PID**를 확인한다.
2. `kill` 로 그 PID에 종료 시그널을 보낸다.
3. `jobs` 는 내가 직접 띄운 백그라운드 작업을 관리하는, 좀 더 좁은 범위의 도구다.
